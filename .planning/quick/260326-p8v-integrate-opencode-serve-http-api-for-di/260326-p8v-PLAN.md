# Plan: Opencode Serve HTTP Integration for Discord Bot

## Overview

Refactor the Discord bot's chat backend to:
1. Auto-spawn `opencode serve` as a subprocess when the bot starts (same service)
2. Connect to it over HTTP (`http://localhost:PORT`)
3. Persist session ID (file-based) for continuous context
4. Reset session after 10 min inactivity
5. Auto-delete bot messages after 30 min

## Tasks

### Task 1: Update `src/bot.py` — Spawn opencode serve and implement HTTP session chat

**Files:** `src/bot.py`

**Action:**
- On startup (in `on_ready`), spawn `opencode serve --port 4096` as a background subprocess via `asyncio.create_subprocess_exec`
- Wait for the server to be ready (poll `GET /` or `GET /session` until 200, max 10s)
- Load or create session ID from `opencode_session.json`
- On `on_message` in `#chat`, call `POST /session/{session_id}/message` with the user prompt
- Parse response `parts[0].text` and reply
- Start two background tasks:
  - `check_inactivity`: every 30s, if no activity for 10 min, create new session
  - `delete_old_messages`: every 10s, delete bot messages older than 30 min
- Remove now-unused `get_ai_response` function and `aiohttp` import (already used — keep it)

**Response parsing:** `data["messages"][-1]["parts"][-1]["text"]` (opencode returns all messages, last is assistant)

**Files:** `src/bot.py`
