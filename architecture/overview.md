# Επισκόπηση Αρχιτεκτονικής (Architecture Overview)

Η αρχιτεκτονική του Orderly έχει σχεδιαστεί γύρω από ένα υβριδικό μοντέλο που εστιάζει στην ταχύτητα, την offline λειτουργία (Local-first) και τη φορολογική συμμόρφωση.

**Βασικοί Πυλώνες:**
1. **Local-first MVP Architecture:** Η λύση βασίζεται σε **Tauri v2+** ως μια εφαρμογή (one-click install) που προσφέρει τοπικό server. Ως βάση δεδομένων χρησιμοποιείται το **Turso (libSQL)** για ταυτόχρονη χρήση στο Cloud με embedded replicas (τοπικά). Η αρχιτεκτονική αυτή υποστηρίζει offline λειτουργικότητα και εξαλείφει την ανάγκη για πολύπλοκες ρυθμίσεις δικτύου από τους ιδιοκτήτες καταστημάτων.
2. **Συγχρονισμός και Ασφάλεια:** Τα δεδομένα συγχρονίζονται στο cloud, υποστηρίζοντας SSE updates για παραγγελίες και ασφαλή καταγραφή IP.
3. **Φορολογική Συμμόρφωση:** Το Phase 1 (MVP) λειτουργεί μόνο ως ordering layer, ενώ το Phase 2 εισάγει API integrations με το υπάρχον POS.
